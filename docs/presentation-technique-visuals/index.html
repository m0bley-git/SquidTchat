Idée 5

Anatomie du protocole JSON

Très utile si tu veux expliquer pourquoi le protocole est simple à étendre. Tu peux basculer entre plusieurs messages réels du projet.

Protocole applicatif auth/register 
{
  "type": "auth/register",
  "timestamp": "2026-04-04T15:00:00Z",
  "payload": {
    "pseudo": "Jean"
  }
}
type

Détermine l'action à exécuter côté serveur ou client.

timestamp

Permet d'horodater les échanges et de tracer les événements.

payload

Contient les données métier: pseudo, contenu, destinataire ou liste d'utilisateurs.
